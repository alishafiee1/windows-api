# طراحی فنی – سرویس محلی دسترسی به خط فرمان ویندوز

---

## معماری کلی

سیستم از سه بخش اصلی درست شده:

1. **سرویس محلی (بک‌اند):** یه برنامه سبک که روی همین کامپیوتر اجراست، پورت `4001` رو گوش می‌ده، دستورات رو دریافت می‌کنه و توی PowerShell اجرا می‌کنه.
2. **لایه امنیت (فیلتر دستورات):** یه فایل لیست که دستورات خطرناک رو نگه می‌داره؛ هر دستور ورودی قبل از اجرا با این لیست چک می‌شه.
3. **رابط تأیید (وب UI):** یه صفحه ساده مرورگری که وقتی دستور خطرناکی دریافت شد، به کاربر نشون داده می‌شه تا تأیید یا رد کنه.

---

## فناوری‌هایی که استفاده می‌کنیم

| فناوری | توضیح ساده |
|--------|------------|
| Python + Flask | برنامه‌ای که روی سرور محلی اجراست و درخواست‌های GET و POST رو مدیریت می‌کنه |
| subprocess (Python) | ماژولی که دستورات رو داخل PowerShell اجرا می‌کنه و خروجی رو برمی‌گردونه |
| JSON | فرمت ساده‌ای که دستور و نتیجه باهاش رد و بدل می‌شه |
| HTML/JS (ساده) | صفحه تأیید که توی مرورگر باز می‌شه — فقط یه دکمه تأیید و رد |
| فایل متنی (blocklist) | فایلی که لیست دستورات ممنوع یا نیازمند تأیید رو نگه می‌داره |

## جریان داده

### حالت معمولی (دستور مجاز)

```
دستیار هوش مصنوعی
    ↓  POST /run  { "cmd": "dir C:\\Users" }
سرویس محلی
    ↓  چک لیست دستورات خطرناک
    ↓  دستور مجازه → اجرا در PowerShell
    ↓  دریافت خروجی
    ↑  { "status": "ok", "output": "..." }
دستیار هوش مصنوعی
```

### حالت دستور خطرناک (نیاز به تأیید)

```
دستیار هوش مصنوعی
    ↓  POST /run  { "cmd": "format C:" }
سرویس محلی
    ↓  تشخیص: این دستور توی blocklist هست
    ↓  ذخیره موقت دستور با یه شناسه (token)
    ↑  { "status": "pending", "token": "abc123", "confirm_url": "http://127.0.0.1:4001/confirm/abc123" }
دستیار هوش مصنوعی → به کاربر می‌گه: "نیاز به تأییده — این لینک رو باز کن"
کاربر لینک رو توی مرورگر باز می‌کنه
    ↓  صفحه تأیید نمایش داده می‌شه (دستور + دکمه‌های تأیید/رد)
    ↓  کاربر کلیک می‌کنه
سرویس محلی
    ↓  اگه تأیید شد → اجرا در PowerShell
    ↑  نتیجه به صفحه مرورگر نمایش داده می‌شه
```

## اندپوینت‌های سرویس

| مسیر | متد | کارش چیه |
|------|-----|-----------|
| `/` | GET | هدایت خودکار به `/help` |
| `/help` | GET | راهنمای JSON؛ شامل `for_ai_assistants` و `examples` |
| `/run` | GET | اجرای دستور با query: `?cmd=...` (مناسب تست مرورگر) |
| `/run` | POST | اجرای دستور با JSON body `{"cmd":"..."}` (پیشنهادی برای AI) |
| `/confirm/{token}` | GET | صفحه تأیید دستور خطرناک رو نمایش می‌ده |
| `/confirm/{token}` | POST | تأیید یا رد کاربر رو دریافت می‌کنه |
| `/blocklist` | GET | لیست دستورات نیازمند تأیید رو برمی‌گردونه |

## فایل blocklist

یه فایل متنی ساده که هر خط یه کلمه کلیدی یا الگو داره. اگه دستور ورودی شامل هر کدوم از این‌ها بشه، قبل از اجرا باید تأیید کاربر گرفته بشه:

**نمونه محتوا:**

```
format
diskpart
del /f /s /q
rmdir /s /q
reg delete
bcdedit
takeown
icacls
netsh advfirewall
shutdown /r /f
```

این فایل قابل ویرایشه — کاربر می‌تونه کلمه اضافه یا حذف کنه.

## متغیرهای محیطی مهم

| متغیر | کاربرد |
|--------|---------|
| `PORT` | شماره پورتی که سرویس گوش می‌ده (پیش‌فرض: `4001`) |
| `BLOCKLIST_PATH` | مسیر فایل لیست دستورات خطرناک |
| `CONFIRM_TIMEOUT` | مدت زمان (ثانیه) که توکن تأیید معتبره، بعدش منقضی می‌شه |
| `SHELL` | نوع محیط اجرا — پیش‌فرض PowerShell |

## مدیریت چرخه سرور

ماژول `core/server_lifecycle.py` مسئول این کارهاست:

- هنگام **شروع**: پروسه قبلی (از `data/server.pid`) و هر listener روی همان پورت بسته می‌شه
- هنگام **خروج** (`Ctrl+C` یا بستن ترمینال): فایل PID پاک می‌شه و پورت آزاد می‌شه
- جلوگیری از اجرای همزمان چند سرور روی یک پورت

---

## محدودیت‌های طراحی

- سرویس فقط روی `127.0.0.1` گوش می‌ده — از شبکه خارجی قابل دسترس نیست
- هر دستور یه timeout داره — اگه بیشتر از حد مجاز طول کشید، کِشته می‌شه و خطا برمی‌گرده
- توکن تأیید یک‌بار مصرفه — بعد از تأیید یا رد شدن، دیگه قابل استفاده نیست
- خروجی دستورات طولانی از یه حد مشخص کوتاه می‌شه تا جواب مناسب برای هوش مصنوعی بمونه
- GET برای دستورات پیچیده محدودیت دارد — AI باید از POST استفاده کند
